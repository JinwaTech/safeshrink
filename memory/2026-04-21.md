# SafeShrink QCalendarWidget 列宽修复 — 2026-04-21

## 问题
历史记录筛选的日期控件日历弹出层，"周一"等星期标签只显示"周..."（被截断）。

## 根因分析过程

1. **错误的假设**：认为星期标签是 QLabel 子控件 → `cw.findChildren(QLabel)` 返回 0 个结果
2. **正确发现**（通过 cal_debug.txt 输出）：
   - 日历只有 **1 个 QTableView**（`name=qt_calendar_calendarview`）
   - 星期标签在 `QTableView.horizontalHeader()` 的 7 个 sections 中
   - 初始列宽只有 24-25 像素（ResizeMode.Stretch）
   - 无 QLabel 子控件

## 最终修复

### 1. eventFilter 重构（history_tab.py）
- **原错误代码**（QLabel 方式）：找不到任何 QLabel，不生效
- **正确代码**（QHeaderView 方式）：

```python
def eventFilter(self, obj, event):
    from PySide6.QtCore import QEvent
    if event.type() != QEvent.Type.Show:
        return super().eventFilter(obj, event)
    cw = None
    for de in (self.filter_date_from, self.filter_date_to):
        if de.calendarWidget() is obj:
            cw = obj
            break
    if not cw:
        return super().eventFilter(obj, event)
    tv = cw.findChild(QTableView, "qt_calendar_calendarview")
    if tv:
        hh = tv.horizontalHeader()
        hh.setSectionResizeMode(QHeaderView.ResizeMode.Fixed)
        for i in range(min(hh.count(), 7)):
            hh.resizeSection(i, 40)  # 40px/列 × 7 = 280px
    return super().eventFilter(obj, event)
```

### 2. 安装位置修正
- 必须在 `calendarWidget()` 本身上安装 filter（不是 dateEdit）
- calendarWidget 由 dateEdit 内部创建（lazy），需先获取再装

```python
cw = de.calendarWidget()
if cw:
    cw.installEventFilter(self)  # ← 装在日历popup上
```

### 3. 初始日期 2020→2026
- `_create_filter_bar()` 和 `_reset_filter()` 中 6 处 `2020` → `2026`
