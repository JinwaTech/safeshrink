# 2026-04-18 SafeShrink Bug 修复总结

## 2026-04-18 下午补充修复

### Bug 6｜xlsx 脱敏时报 "Cell is read only"（第二次修复迭代）
- **现象**: xlsx 脱敏时报 `Cell is read only`
- **根因**: `_sanitize_native_xlsx` 复用 `_native_doc`（由 `read_only=True` 的 workbook 返回），只读 workbook 不能修改单元格
- **修复**: `_sanitize_native_xlsx` 不复用只读 workbook，自己用 `read_only=False` 重新加载可写副本

### Bug 7｜xlsx 预览和脱敏后合计栏消失（第三次修复迭代）
- **现象**: 公式单元格（如 `=SUM(B2:B5)`）显示的"合计"数字全部消失，变成空白
- **根因**: `load_workbook(path, read_only=True, data_only=True)` —— data_only=True 时，无缓存值的公式返回 None，导致整行消失
  - `data_only=True`：只读单元格的值（上次保存时 Excel 计算的结果），没有缓存时为 None
  - `read_only=True`：`data_only` 参数在只读模式下无效，始终返回缓存值
  - 两者叠加：所有公式单元格都变成 None
- **修复**: 去掉 `read_only=True` 和 `data_only=True`，改为 `load_workbook(path, data_only=False)`，预览时显示公式文字，脱敏时保留公式（Excel 重新打开时自动重算）

### 关键教训
openpyxl 的 `data_only` 和 `read_only` 参数互相影响：
- `read_only=True` → `data_only` 参数无效，始终返回缓存值
- `data_only=True` → 公式返回缓存值（无缓存则为 None），不返回公式文字
- `data_only=False` → 返回公式文字（作为字符串），不返回计算结果
- `read_only=False` → 可写，但内存占用更大
